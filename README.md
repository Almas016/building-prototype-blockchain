# Введение
Блокчейн — это просто распределенная база данных записей. Но что делает ее уникальной, так это то, что это не частная база данных, а публичная, т.е. каждый, кто ею пользуется, имеет ее полную или частичную копию. И новая запись может быть добавлена ​​только с согласия других хранителей базы данных. 

Тут реализована упрощенная криптовалюта, основанная на простой реализации блокчейна.

# Блок
В блоках хранится ценная информация. Например, транзакции, его версия, текущая метка времени и хэш предыдущего блока. Упрощенная версия, содержащую только важную информацию. Вот как это выглядит:
```go
type Block struct {
	Timestamp     int64  // время когда был создан блок
	Data          []byte // фактическая ценная информация
	PrevBlockHash []byte // хэш предыдущего блока
	Hash          []byte // хэш блока
}
```

# Блокчейн
Блокчейн —  это упорядоченный список с обратными ссылками. Это означает, что блоки хранятся в порядке вставки и что каждый блок связан с предыдущим.

Для прототипа блокчейна был использован массив, так как нужен был упорядоченный список хэшей.
```go
type Blockchain struct {
	blocks []*Block
}
```
Чтобы добавить новый блок, нужен существующий блок, но в цепочке блоков нет блоков. В любом блокчейне должен быть хотя бы один блок, и такой блок, первый в цепочке, называется блоком генезиса.
